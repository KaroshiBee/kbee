# AN231E04 quick reference

Scratch notes — fill in as we go.

## Chip at a glance

- Dynamically reconfigurable FPAA
- **4 Configurable Analog Blocks (CABs)** in a 2x2 grid
- Switched-capacitor core; master clock typically 16 MHz
- 4 programmable I/O cells + 2 output cells (depending on AD2 configuration)
- On-chip 256-byte LUT for waveform / arbitrary transfer function generation
- Serial configuration interface (primary + secondary); dynamic reconfig
  supported while the design is running

## Typical clocks

- `ClockA` derived from master (e.g. 250 kHz for audio-rate SC filters)
- Each CAM has a primary and sometimes secondary clock selection
- Mind Nyquist: SC filter corner \<< clock / 2

## Design flow (AnadigmDesigner2)

1. Create new circuit, pick AN231E04.
1. Drop CAMs (filters, gain, SumDiff, rectifier, etc.) from the library.
1. Wire to chip I/O cells; configure I/O type (differential, bypass,
   chopper-stabilized, etc.).
1. Simulate (AD2 has a built-in time/freq sim).
1. Compile -> generates a configuration bitstream.
1. Either download live over serial to the dev board, or export a C
   header (`*.h`) of the primary config + any dynamic updates and ship
   that to an MCU.

## Dev board

AN231K04 evaluation board: USB-serial bridge to the chip's config port.
AD2 talks to it directly when the board is attached.

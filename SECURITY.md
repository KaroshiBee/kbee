# Security

## Reporting

If you believe you have found a security issue in KBee software, tooling, or
published documentation:

1. Open an issue at
   [codeberg.org/Karoshibee/kbee/issues](https://codeberg.org/Karoshibee/kbee/issues)
   with the prefix `[security]` in the title, **or**
1. Contact the repository maintainers via Codeberg if you need to share details
   privately before public disclosure.

Please include enough information to reproduce the problem (affected paths,
commands, and expected vs actual behaviour).

## Scope

This repository contains reference algorithms, FPAA/ASIC design artefacts, and
build guides. It does not ship firmware, hosted services, or network-facing
components. Reports about proprietary third-party tools (AnadigmDesigner2,
AN231E04) should go to their vendors unless the issue is in our published
`.ad2` design or documentation.

## Response

We aim to acknowledge reports within a reasonable time and will coordinate on
disclosure timing for confirmed issues.
